## 2. 容器分解 (Container Diagram)

系统由 4 个运行时容器组成：

| 容器 | 技术 | 职责 |
|------|------|------|
| **Frontend** | React 18 + TypeScript + Vite | UI 渲染，支持 Electron 壳和 Web 壳 |
| **Electron Shell** | Electron 33 | 桌面端壳层：原生窗口、文件系统、IPC、自动更新 |
| **Backend API** | Python 3.12 + FastAPI | REST API、认证、数据验证、任务编排 |
| **Celery Worker** | Python 3.12 + Celery 5 | 异步算法执行 (音高检测 → MIDI → 乐谱) |

基础设施：

| 组件 | 技术 | 职责 |
|------|------|------|
| 数据库 | PostgreSQL 16 | 持久化用户、项目、文件、任务元数据 |
| 缓存 | Redis 7 | Celery broker、session 缓存、实时推送 |
| 文件存储 | 本地磁盘 / MinIO / S3 | 存储音频文件、MIDI、乐谱 |
| 反向代理 | Nginx | TLS 终端、静态文件服务、WebSocket 代理 |

## 4. 端到端数据流

### 4.1 哼唱转旋律 — 关键路径

```
用户录音 (WAV)
    │
    ▼
[Frontend] RecordPage.tsx
    │  MediaRecorder → Blob
    │
    ▼
[Frontend] API Client
    │  POST /api/v1/projects/{id}/files  (multipart)
    │  { filename: "hum_xxx.wav", category: "recording", ... }
    │
    ▼
[Backend] api/files.ts  →  file_service.py  →  storage.save()
    │  存储到文件系统 /data/{yyyymm}/{uuid}.wav
    │  写入 file 表记录
    │  返回 file_id
    │
    ▼
[Frontend] 收到 file_id
    │  POST /api/v1/tasks/melody
    │  { file_id, instrument: 1, is_drum: true, is_bass: true, ... }
    │
    ▼
[Backend] api/tasks.ts → task_service.py → 创建 task 记录 (status=pending)
    │  推送 task_id → Celery: humming_to_melody.delay(task_id=..., file_id=...)
    │
    ▼
[Celery Worker] tasks.py::humming_to_melody()
    │  ┌─────────────────────────────────────────────┐
    │  │  1. AudioLoader: load WAV → AudioBuffer      │
    │  │  2. PitchDetector: detect → [(time, freq,    │
    │  │     confidence), ...]                        │
    │  │  3. BeatTracker: detect → BPM, beat_times   │
    │  │  4. Quantizer: pitch + beat → Note[]        │
    │  │  5. MidiGenerator: Note[] → PrettyMIDI      │
    │  │  6. ChordDetector: Analyze harmony           │
    │  │  7. AccompanimentGenerator: Drum + Bass +    │
    │  │     Chord MIDI tracks                        │
    │  │  8. MidiMerger: merge → final MIDI file      │
    │  │  9. save output file → file_id               │
    │  │ 10. update task (status=completed)            │
    │  └─────────────────────────────────────────────┘
    │
    ▼
[Backend] tasks 表更新 → status=completed, result_file_id=xxx
    │  WebSocket 推送 /ws/tasks/{task_id} → {event: "completed"}
    │
    ▼
[Frontend] useTaskPolling() 收到 completed 事件
    │  显示完成状态
    │  加载乐谱 /scores/{id} → 渲染 VexFlow
    │  加载音频 → 显示播放器
```


## 5. 模块间依赖规则

### 5.1 单向依赖链

```
API Layer (薄路由)
    │ 只调用 Service
    ▼
Service Layer (纯函数)
    │ 调用 Storage、Algorithms、触发 Worker
    ▼
Algorithm Layer (纯函数，无 I/O)
    │ 只依赖 schemas 中的数据类
    ▼
Infrastructure (DB、Redis、文件系统)
```

### 5.2 禁止反向依赖

- API 层**不能**直接操作数据库
- Service 层**不能**直接返回 ORM 模型 (必须通过 Schema 转换)
- Algorithm 层**不能**感知 HTTP、数据库或任何副作用
- Storage 层**不能**导入 API 或 Service 层的任何内容

### 5.3 跨模块通信协议

- 模块间仅通过**纯数据类** (Pydantic / dataclass) 通信
- 不允许传递 ORM 实例或 DB Session 跨模块
- 每个 Service 函数签名：`(deps, input_dto) → Result[Error, OutputDto]`

## 6. 错误处理策略

### 6.1 Result 类型 (核心代数类型)

```python
# app/core/result.py
from __future__ import annotations
from dataclasses import dataclass
from typing import Generic, TypeVar, Union

T = TypeVar("T")
E = TypeVar("E")


@dataclass(frozen=True)
class Ok(Generic[T]):
    value: T


@dataclass(frozen=True)
class Err(Generic[E]):
    error: E


Result = Union[Ok[T], Err[E]]
# 使用时：Result[User, AppError]
```

### 6.2 Service 层不使用异常

```python
# ✅ Service 函数签名示例
def create_user(db: AsyncSession, data: CreateUserDto) -> Result[User, AppError]:
    """纯函数：要么返回 Ok(User)，要么返回 Err(AppError)。不抛异常。"""

# ❌ 禁止的写法
def create_user(db, data) -> User:
    """这可能会抛 IntegrityError 或 ValidationError —— 允许但不够优雅。"""
```

### 6.3 异常仅在框架边界捕获

- FastAPI 异常处理器 (Exception Handler) 在 API 层将 Exception 转换为 Err
- Celery 自动处理 Worker 抛出的异常 → status=failed
- ORM / HTTP 客户端异常 → catch → 封装为 Err → 不传播原始异常

## 7. 关键架构决策记录 (ADR)

| ID | 决策 | 理由 |
|----|------|------|
| ADR-001 | FastAPI 替代 Spring Boot + Django | 统一 Python 栈，自动 OpenAPI，异步原生 |
| ADR-002 | Celery + Redis 替代 thread | 异步非阻塞，可水平扩展 Worker，状态持久化 |
| ADR-003 | CREPE 替代 librosa chroma_cqt | 连续音高 (非 12 音级量化)，更高精度 |
| ADR-004 | PostgreSQL 替代 MySQL + SQLite | JSONB 灵活扩展，全文搜索，异步驱动 |
| ADR-005 | VexFlow 5 替代手写画布 | 成熟开源，Canvas/SVG 双渲染，React 友好 |
| ADR-006 | Electron + Web 共享 95% 代码 | 维护一套代码，两种壳 |
| ADR-007 | Zustand 替代 Redux | 无 boilerplate，TypeScript 原生支持 |
| ADR-008 | shadcn/ui 替代 Ant Design | 可定制，Tailwind 样式，无运行时依赖 |
| ADR-009 | Pydantic v2 数据契约 | FastAPI 内置验证，高性能 Rust 核心 |
| ADR-010 | Result 类型替代异常传播 | 纯函数范式，类型安全，调用者必须处理错误 |
