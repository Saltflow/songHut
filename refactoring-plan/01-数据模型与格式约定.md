# SongHut 2.0 — 数据模型与格式约定

## 1. 设计原则

1. **一切显式化**：每个数据类的定义即文档。不依赖隐式约定。
2. **不可变优先**：所有数据类 `frozen=True` (Python) / `readonly` (TypeScript)。修改返回新实例。
3. **类型完备**：不使用 `dict` 或 `Any` 传递业务数据。每个数据格式都有类型定义。
4. **代数类型驱动**：用 `Union[Ok[T], Err[E]]` 替代异常，用 `Optional[T]` 明确表示可空。
5. **单一真实来源**：Pydantic schema = API 契约 = 内部表示。不维护多套模型。

## 3. 算法数据格式

### 3.1 音频数据

```python
from dataclasses import dataclass
import numpy as np


@dataclass(frozen=True)
class AudioBuffer:
    """音频数据的统一表示。算法层只处理此类型，不关心文件格式。"""
    samples: np.ndarray           # shape: (n_samples,) 或 (n_channels, n_samples)
    sample_rate: int              # Hz, 如 44100
    n_channels: int               # 1 (mono) 或 2 (stereo)
    duration_sec: float           # 计算值: samples / sample_rate

    @property
    def mono(self) -> "AudioBuffer":
        """混音到单声道"""
        if self.n_channels == 1:
            return self
        return AudioBuffer(
            samples=np.mean(self.samples, axis=0),
            sample_rate=self.sample_rate,
            n_channels=1,
        )
```

```python
from enum import IntEnum


class NoteName(IntEnum):
    """音名转 MIDI 偏移量 (C=0)"""
    C = 0
    C_SHARP = 1
    D = 2
    D_SHARP = 3
    E = 4
    F = 5
    F_SHARP = 6
    G = 7
    G_SHARP = 8
    A = 9
    A_SHARP = 10
    B = 11


@dataclass(frozen=True)
class Note:
    """量化后的乐谱音符。不只是一个 MIDI 事件——包含乐理信息。"""
    pitch_midi: int              # MIDI 音高数 (0-127), 如 60 = C4
    note_name: NoteName          # 音名
    octave: int                  # 八度, 如 4
    onset_sec: float             # 起始时间 (秒)
    duration_sec: float          # 持续时间
    velocity: int                # 力度 (0-127)
    is_rest: bool
    beat_position: float         # 在小节中的位置 (拍数, 如 1.5 = 第1拍半)
    measure: int                 # 所属小节编号 (从 0 或 1)
```

### 3.5 和弦

```python
@dataclass(frozen=True)
class Chord:
    """和声/和弦分析结果"""
    root: NoteName               # 根音
    quality: str                 # "maj", "min", "dim", "aug", "dom7", "maj7", "min7", "sus4", etc.
    bass_note: NoteName          # 最低音 (转位时与 root 不同)
    onset_sec: float
    duration_sec: float
    notes: tuple[int, ...]       # 构成该和弦的 MIDI 音高集合

    @property
    def shorthand(self) -> str:
        """简洁和弦标记: "C", "Dm", "G7", "Am" 等"""
        root_name = ["C", "C#", "D", "D#", "E", "F", "F#", "G", "G#", "A", "A#", "B"]
        quality_map = {
            "maj": "", "min": "m", "dim": "dim", "aug": "aug",
            "dom7": "7", "maj7": "M7", "min7": "m7", "dim7": "dim7",
            "sus4": "sus4",
        }
        return f"{root_name[int(self.root)]}{quality_map.get(self.quality, '')}"
```

## 5. 命名约定

### 5.1 通用

| 类别 | 约定 | 示例 |
|------|------|------|
| 数据库表 | 小写复数 snake_case | `users`, `project_members` |
| ORM 模型类 | PascalCase 单数 | `class User`, `class FileRecord` |
| DTO/Schema 类 | PascalCase + `Dto` 后缀 | `class UserDto`, `class CreateFileDto` |
| Service 函数 | snake_case + 动词开头 | `create_user()`, `get_project_by_id()` |
| API 路由函数 | snake_case | `async def list_projects()` |
| 枚举类 | PascalCase | `class FileCategory`, `class TaskStatus` |
| 枚举值 | UPPER_SNAKE_CASE | `MELODY`, `PROCESSING` |
| URL 路径 | kebab-case | `/api/v1/project-members` |
| JSON 字段 | camelCase | `fileId`, `createdAt` (FastAPI 自动转换) |

### 5.2 文件命名

```
# Python 模块
algorithms/pitch/detector.py    # 蛇形命名，简短
algorithms/pitch/protocol.py    # Protocol 接口文件固定用 protocol.py

# TypeScript 模块
features/recording/RecordPage.tsx        # PascalCase 组件
features/recording/useAudioRecorder.ts   # useXxx hooks
stores/auth.store.ts                    # 以 .store.ts 结尾
api/client.ts                          # API 客户端
```

### 5.3 数据流命名

```
AudioBuffer          → 算法层的音频输入
PitchFrame           → 音高检测中间结果
Note                 → 量化后音符 (乐谱逻辑单位)
MidiTrackData        → MIDI 轨道 (物理存储单位)
MidiOutput           → 完整的 MIDI 文件
ScoreDto             → 乐谱元数据 (数据库)
RenderedNote         → 前端渲染用音符数据 (含坐标)
```

## 6. 数据格式转换 Pipeline

### 6.1 文件格式映射

```
输入音频格式:
  WAV (.wav)          ← 首选, 无损
  MP3 (.mp3)          ← 可选, 有损
  M4A (.m4a)          ← Apple 设备
  WebM (.webm)        ← Web 录制
  ↓ 统一转码
  WAV 16-bit 44.1kHz mono   ← 算法统一输入格式

输出音频格式:
  MIDI (.mid)         ← 标准 MIDI 文件, Type 1 (多轨道)
  MusicXML (.xml)     ← 乐谱数据交换格式
  WAV (.wav)          ← 可选的音频预览 (FluidSynth 合成)
```

### 6.2 类型映射规则

```
Python Pydantic           TypeScript (前端)
────────────────────      ────────────────────
UUID                      string (UUID formatted)
datetime                  string (ISO 8601)
int                       number
float                     number
str                       string
bool                      boolean
list[T]                   T[]
dict[str, Any]            Record<string, any>
Optional[T]               T | null
Enum → str                Union of string literals
```

### 6.3 时间精度约定

```
音频时间戳:               float, 单位秒, 精度 1ms (0.001)
MIDI tick:               int, 单位 tick (480 每四分音符)
音符时长:                 float 秒 (存储) / str 时值 (VexFlow: "q", "8.")
BPM:                     int (很少用浮点 BPM)
乐谱拍号:                 str ("4/4", "3/4", "6/8")
旋律位置 (小节内):        float (数字表示, 如 1.5 = 第1拍半)
```
